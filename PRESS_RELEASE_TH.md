# ข่าวประชาสัมพันธ์: แพลตฟอร์ม AWS Code-Server สำหรับทีมพัฒนาซอฟต์แวร์

**เผยแพร่ทันที**

---

## แพลตฟอร์มพัฒนาซอฟต์แวร์บนคลาวด์พร้อม AI ระดับองค์กร ลดต้นทุน 60% สำหรับทีมขนาดเล็ก

**กรุงเทพฯ ประเทศไทย** – ทีมพัฒนาซอฟต์แวร์ที่ประสบปัญหาค่าใช้จ่ายสูงและความซับซ้อนของสภาพแวดล้อมการพัฒนาบนคลาวด์มีทางเลือกใหม่แล้ว แพลตฟอร์ม AWS Code-Server Multi-Developer ที่เปิดตัววันนี้ นำเสนอสภาพแวดล้อม VS Code บนเบราว์เซอร์พร้อมการช่วยเหลือจาก Claude AI ในราคาเพียง 52-56 ดอลลาร์สหรัฐต่อนักพัฒนาต่อเดือนสำหรับค่าโครงสร้างพื้นฐาน – ต่ำกว่าครึ่งหนึ่งของราคาโซลูชันเดสก์ท็อปคลาวด์แบบดั้งเดิม

แพลตฟอร์มที่สร้างบนโครงสร้างพื้นฐาน AWS และใช้ประโยชน์จากโมเดล Claude ของ AWS Bedrock สามารถแก้ปัญหาสำคัญสองประการของทีมพัฒนา ได้แก่ การจัดการเดสก์ท็อปคลาวด์ราคาแพงสำหรับนักพัฒนาแต่ละคน และการประสานงานการสมัครใช้งาน AI ผู้ช่วยแยกกันสำหรับนักพัฒนาแต่ละคน การติดตั้งครั้งเดียวรองรับสภาพแวดล้อมการพัฒนาที่แยกออกจากกันได้ถึงแปดชุดบนเซิร์ฟเวอร์ AWS EC2 เครื่องเดียว พร้อมด้วยใบรับรอง HTTPS อัตโนมัติ พื้นที่จัดเก็บข้อมูลถาวร ระบบติดตามสถานะ และการสำรองข้อมูลรายวัน

"เราพบปัญหาเดิมซ้ำๆ" ทีมวิศวกรของแพลตฟอร์มอธิบาย "ทีมต่างๆ ต้องการนำ AI มาช่วยในการพัฒนา แต่ตัวเลขไม่คุ้มค่า เดสก์ท็อปคลาวด์แต่ละเครื่องเสียค่าใช้จ่าย 100-200 ดอลลาร์สหรัฐต่อเดือนต่อนักพัฒนา เพิ่มการสมัครใช้ Claude API อีกคนละ 20-100 ดอลลาร์สหรัฐ รวมแล้วเป็น 120-300 ดอลลาร์สหรัฐต่อนักพัฒนาก่อนเริ่มเขียนโค้ดบรรทัดแรกด้วยซ้ำ สำหรับทีมแปดคน ต้องเสียเงิน 960-2,400 ดอลลาร์สหรัฐต่อเดือน หลายทีมไม่สามารถอธิบายความคุ้มค่าได้"

แพลตฟอร์มใหม่นี้เปลี่ยนสมการดังกล่าว ค่าโครงสร้างพื้นฐานอยู่ที่ประมาณ 420-450 ดอลลาร์สหรัฐต่อเดือนสำหรับนักพัฒนาแปดคน – หรือประมาณ 52-56 ดอลลาร์สหรัฐต่อนักพัฒนา การใช้งาน AWS Bedrock เรียกเก็บเงินรวมกันตามการใช้ AI จริง โดยไม่มีค่าธรรมเนียมรายหัว สำหรับทีมที่ใช้ AI ในระดับปานกลาง ต้นทุนรวมทั้งหมดรวมถึงการเรียกใช้ Claude API โดยทั่วไปอยู่ในช่วง 180-240 ดอลลาร์สหรัฐต่อนักพัฒนาต่อเดือน แสดงให้เห็นการประหยัดต้นทุน 40-60% เมื่อเทียบกับโซลูชันแบบดั้งเดิม

นวัตกรรมสำคัญของแพลตฟอร์มคือการผสานรวม AWS Bedrock อย่างไร้รอยต่อ ต่างจากโซลูชันที่ต้องใช้ API key และการสมัครใช้งานแยกกัน นักพัฒนาสามารถยืนยันตัวตนผ่าน AWS IAM role โดยอัตโนมัติ หัวหน้าทีมได้รับการเรียกเก็บเงินแบบรวมศูนย์และสามารถติดตามการใช้ AI ของนักพัฒนาแต่ละคนผ่าน CloudWatch logs แก้ไขทั้งปัญหาการจัดการต้นทุนและความท้าทายด้านความปลอดภัยที่ทำให้การนำ AI มาใช้ในทีมพัฒนาช้าลง

การติดตั้งใช้เวลาประมาณสามสิบนาทีโดยใช้ AWS Cloud Development Kit (CDK) ในภาษา Python ทีมกำหนดค่าสามอย่างในไฟล์คอนฟิก – ชื่อโดเมน, Route53 hosted zone ID และ IP ของผู้ดูแลระบบ – จากนั้นรันคำสั่งเดียว การติดตั้งอัตโนมัติสร้างโครงสร้างพื้นฐานแบบครบวงจร: เครือข่าย VPC, เซิร์ฟเวอร์ EC2, Application Load Balancer พร้อม SSL termination, บันทึก DNS ของ Route53, การติดตามสถานะด้วย CloudWatch และตารางการสำรองข้อมูลด้วย AWS Backup

แพลตฟอร์มแก้ไขความท้าทายเฉพาะภูมิภาคสำหรับทีมในเอเชียตะวันออกเฉียงใต้ AWS Bedrock ซึ่งเป็นบริการจัดการสำหรับ Claude และ AI model อื่นๆ ของ Amazon ยังไม่มีในภูมิภาคกรุงเทพฯ (ap-southeast-7) แพลตฟอร์มติดตั้งโครงสร้างพื้นฐานในกรุงเทพฯ เพื่อความหน่วงต่ำสำหรับนักพัฒนาในท้องถิ่น พร้อมทั้งกำหนดเส้นทางการเรียกใช้ Claude API ไปยังภูมิภาคสิงคโปร์ (ap-southeast-1) โดยอัตโนมัติ ซึ่งเป็นสถานที่ที่มี Bedrock ใกล้ที่สุด ความหน่วงข้ามภูมิภาคเพิ่มเพียง 20-50 มิลลิวินาทีต่อคำขอ AI

"ก่อนติดตั้งแพลตฟอร์มนี้ เราเสียเงิน 150 ดอลลาร์สหรัฐต่อนักพัฒนาต่อเดือนสำหรับเดสก์ท็อปคลาวด์ บวกกับการสมัครใช้ Claude API แยกกันตั้งแต่ 20 ถึง 100 ดอลลาร์สหรัฐขึ้นอยู่กับการใช้งาน" CTO ของบริษัทซอฟต์แวร์ขนาดกลางในกรุงเทพฯ ที่เข้าร่วมการทดสอบแพลตฟอร์มในช่วงแรก รายงาน "ตอนนี้เราใช้นักพัฒนาแปดคนด้วยค่าใช้จ่ายโครงสร้างพื้นฐานไม่ถึง 500 ดอลลาร์สหรัฐต่อเดือน ต้นทุน Bedrock ปรับตามการใช้งานจริง และเราสามารถดูและควบคุมค่าใช้จ่าย AI ได้ในที่สุด การติดตั้งใช้เวลาแค่บ่ายเดียวแทนที่จะเป็นหลายสัปดาห์ของงาน DevOps"

นักพัฒนาแต่ละคนได้รับซับโดเมนของตัวเอง – dev1 ถึง dev8.yourdomain.com – พร้อมใบรับรอง SSL อัตโนมัติผ่าน AWS Certificate Manager Code-server ซึ่งเป็นสภาพแวดล้อม VS Code บนเบราว์เซอร์แบบโอเพนซอร์สจาก Coder ทำงานในคอนเทนเนอร์ Docker แยกกันพร้อมขีดจำกัด CPU และหน่วยความจำเฉพาะ พื้นที่ทำงานจัดเก็บบนไดรฟ์ EBS ขนาด 500GB พร้อมสแน็ปช็อตอัตโนมัติรายวันเก็บไว้สามสิบวัน

Claude Code extension ซึ่งเป็นส่วนขยาย VS Code อย่างเป็นทางการของ Anthropic กำหนดค่าอัตโนมัติเพื่อใช้จุดปลายทาง (endpoint) ของ AWS Bedrock ในสิงคโปร์ นักพัฒนาตั้งค่าโมเดล Claude ที่ต้องการ – Haiku สำหรับงานเร็วที่ 0.25 ดอลลาร์สหรัฐต่อหนึ่งล้านโทเค็นอินพุต, Sonnet สำหรับประสิทธิภาพสมดุลที่ 3 ดอลลาร์สหรัฐต่อหนึ่งล้านโทเค็นอินพุต หรือ Sonnet 4.5 รุ่นล่าสุดสำหรับการใช้เหตุผลที่ซับซ้อน การยืนยันตัวตนเกิดขึ้นโดยโปร่งใสผ่าน IAM role ของเซิร์ฟเวอร์ EC2 ไม่มีการจัดเก็บหรือจัดการ API key

การติดตามสถานะด้วย CloudWatch ให้ภาพรวมของสุขภาพระบบ สถานะคอนเทนเนอร์ และรูปแบบการใช้ AI การแจ้งเตือนที่กำหนดค่าไว้ล่วงหน้าแจ้งผู้ดูแลระบบเกี่ยวกับการใช้ CPU สูง ปัญหาพื้นที่ดิสก์ หรือการตรวจสอบสถานะล้มเหลว คิวรี CloudWatch Logs Insights สามารถแบ่งการใช้งาน Bedrock ตามนักพัฒนา โมเดล หรือช่วงเวลา ทำให้ทีมเข้าใจและปรับปรุงค่าใช้จ่าย AI ได้

ตลาดเป้าหมายของแพลตฟอร์มคือทีมพัฒนาที่มีสองถึงแปดคนที่ต้องการประโยชน์ของสภาพแวดล้อมการพัฒนาบนคลาวด์โดยไม่ต้องรับมือกับความซับซ้อนและต้นทุนของโซลูชัน VDI ระดับองค์กร สตาร์ทอัพสามารถใช้งานสำหรับทีมกระจาย SMB สามารถมาตรฐานสภาพแวดล้อมการพัฒนาข้ามสำนักงาน ทีมองค์กรสามารถสร้างสภาพแวดล้อมแซนด์บ็อกซ์แยกกันสำหรับโครงการเฉพาะ

รุ่นปัจจุบันรองรับภูมิภาค AWS ที่มีความสามารถ VPC, EC2 และ Bedrock แม้ว่าโครงสร้างพื้นฐานสามารถติดตั้งได้ทุกที่ ทีมต้องกำหนดเส้นทางคำขอ Bedrock ไปยังสิงคโปร์ (ap-southeast-1), โตเกียว (ap-northeast-1) หรือภูมิภาคสหรัฐอเมริกาขึ้นอยู่กับสถานที่ รายการแผนงานในอนาคตรวมถึงการปรับขนาดอัตโนมัติสำหรับขนาดทีมแบบไดนามิก การรองรับเซิร์ฟเวอร์ GPU สำหรับงานแมชชีนเลิร์นนิง การผสานรวมกับ GitHub Actions สำหรับเวิร์กโฟลว์ CI/CD และการนำไปใช้ Terraform สำหรับทีมที่ไม่ใช้ AWS CDK

เอกสารประกอบรวมถึงคู่มือเริ่มต้นใช้งานห้านาที คำแนะนำการติดตั้งครบถ้วน คู่มือการกำหนดค่า Bedrock บทช่วยสอนการติดตามการใช้งาน และขั้นตอนแก้ไขปัญหา แพลตฟอร์มพร้อมใช้งานแล้วในรูปแบบ Infrastructure-as-Code แบบเปิดที่ทีมสามารถปรับแต่ง ควบคุมเวอร์ชัน และขยายตามความต้องการเฉพาะ

สำหรับทีมพัฒนาที่กำลังประเมินแพลตฟอร์มพัฒนาบนคลาวด์ เศรษฐศาสตร์ตรงไปตรงมา โซลูชันเดสก์ท็อปคลาวด์แบบดั้งเดิมเริ่มต้นที่ 100-200 ดอลลาร์สหรัฐต่อนักพัฒนาต่อเดือนสำหรับโครงสร้างพื้นฐานเพียงอย่างเดียว ก่อนเพิ่มความสามารถ AI GitHub Codespaces เรียกเก็บเงินต่อชั่วโมงการใช้งาน ตั้งแต่ 0.18 ถึง 0.72 ดอลลาร์สหรัฐต่อชั่วโมงสำหรับสองถึงแปดคอร์ AWS WorkSpaces ให้เดสก์ท็อป Windows หรือ Linux แบบเต็มรูปแบบที่ 25-75 ดอลลาร์สหรัฐต่อเดือนต่อเดสก์ท็อป แต่ต้องการการผสานรวม AI แยกต่างหาก

แพลตฟอร์ม AWS Code-Server Multi-Developer เสนอต้นทุนโครงสร้างพื้นฐานคงที่ประมาณ 420-450 ดอลลาร์สหรัฐต่อเดือนโดยไม่คำนึงถึงชั่วโมงการใช้งาน รองรับนักพัฒนาแปดคนในสภาพแวดล้อมแยกกันพร้อมการช่วยเหลือ AI แบบบูรณาการ สำหรับทีมที่ใช้สภาพแวดล้อมการพัฒนาเต็มเวลามากกว่าเป็นครั้งคราว เศรษฐศาสตร์ต่อหน่วยเอื้ออำนวยต่อแพลตฟอร์มอย่างมาก

ความสำเร็จของแพลตฟอร์มขึ้นอยู่กับการแก้ปัญหาสามประการพร้อมกัน: ลดต้นทุนโครงสร้างพื้นฐานผ่านการใช้ทรัพยากรร่วมกันอย่างมีประสิทธิภาพ กำจัดการจัดการ API key ผ่านการยืนยันตัวตน IAM และจัดหาการเรียกเก็บเงินและการติดตามการใช้งานแบบรวมศูนย์สำหรับบริการ AI การปรับปรุงแต่ละอย่างจะมีค่าในตัวเอง รวมกัน พวกเขาขจัดอุปสรรคที่ป้องกันไม่ให้ทีมขนาดเล็กนำการพัฒนาที่ได้รับความช่วยเหลือจาก AI มาใช้ในระดับขนาดใหญ่

"คำถามที่เราถามตัวเองคือ: ทำไมทีมของนักพัฒนาแปดคนต้องการเดสก์ท็อปคลาวด์แยกกันแปดเครื่อง การสมัคร Claude แยกกันแปดอัน และมีคนจัดการข้อมูลประจำตัวสิบหกชุดด้วยตนเอง?" ทีมวิศวกรระบุ "เทคโนโลยีที่มีอยู่สามารถทำได้ดีกว่านี้ AWS CDK ทำให้โครงสร้างพื้นฐานสามารถทำซ้ำได้ AWS Bedrock ให้การเข้าถึง Claude แบบจัดการด้วยการยืนยันตัวตน IAM Docker ให้การแยก เราเพียงแค่ต้องประกอบชิ้นส่วนเหล่านี้ในลักษณะที่สมเหตุสมผลทางเศรษฐกิจและการดำเนินงานสำหรับทีมขนาดเล็ก"

ทีมผู้นำการใช้งานในช่วงแรกรายงานว่าการเริ่มต้นใช้งานนักพัฒนาใหม่เร็วขึ้นอย่างมาก แทนที่จะจัดเตรียมเดสก์ท็อปคลาวด์แต่ละเครื่องและแจกจ่าย API key ผู้ดูแลระบบเพียงดึงรหัสผ่านจาก AWS Secrets Manager และแชร์ URL ซับโดเมนของนักพัฒนา นักพัฒนาใหม่เข้าสู่ระบบสภาพแวดล้อม VS Code ที่กำหนดค่าไว้ล่วงหน้าพร้อม Claude Code ที่ตั้งค่าเรียบร้อยแล้ว พื้นที่ทำงานคงอยู่ในเซสชัน และการสำรองข้อมูลรายวันรับประกันความปลอดภัยของข้อมูล

แนวทาง Infrastructure-as-Code แบบเปิดของแพลตฟอร์มหมายความว่าทีมควบคุมการติดตั้งของพวกเขาอย่างสมบูรณ์ พวกเขาสามารถแก้ไขประเภทเซิร์ฟเวอร์ ปรับขีดจำกัดทรัพยากร เพิ่มนักพัฒนาเพิ่มเติม บูรณาการกับ VPC ที่มีอยู่ หรือปรับแต่งเครือข่ายสำหรับข้อกำหนดความปลอดภัยเฉพาะ การนำไปใช้ AWS CDK Python ให้คำจำกัดความโครงสร้างพื้นฐานที่อ่านและบำรุงรักษาได้ที่ทีมสามารถควบคุมเวอร์ชันควบคู่ไปกับโค้ดแอปพลิเคชัน

มองไปข้างหน้า แผนงานของแพลตฟอร์มเน้นการขยายความยืดหยุ่นในขณะที่รักษาข้อเสนอคุณค่าหลักของการพัฒนาบนคลาวด์ที่เรียบง่ายและคุ้มค่าพร้อม AI แบบบูรณาการ การรองรับการปรับขนาดอัตโนมัติจะทำให้ทีมสามารถเพิ่มหรือลดสภาพแวดล้อมนักพัฒนาแบบไดนามิก ตัวเลือกเซิร์ฟเวอร์ GPU จะรองรับทีมที่สร้างโมเดลแมชชีนเลิร์นนิง การผสานรวมกับ AWS SSO จะทำให้การยืนยันตัวตนคล่องตัวสำหรับทีมองค์กรที่มีผู้ให้บริการข้อมูลประจำตัวอยู่แล้ว

สำหรับทีมที่พร้อมติดตั้ง กระบวนการเริ่มต้นด้วยการโคลนพื้นที่เก็บข้อมูล แก้ไขค่าคอนฟิกสามค่า และรันสคริปต์ติดตั้ง ภายในสามสิบนาที โครงสร้างพื้นฐานพร้อมใช้งาน ภายในหนึ่งชั่วโมง นักพัฒนากำลังเขียนโค้ดด้วยความช่วยเหลือ AI ในสภาพแวดล้อม VS Code บนเบราว์เซอร์ คุณค่าของแพลตฟอร์มชัดเจนทันที: ต้นทุนต่ำกว่า การดำเนินงานที่ง่ายขึ้น และการมองเห็นแบบรวมศูนย์ทั้งโครงสร้างพื้นฐานและค่าใช้จ่าย AI

ทีมที่สนใจแพลตฟอร์ม AWS Code-Server Multi-Developer สามารถเข้าถึงซอร์สโค้ดที่สมบูรณ์ เอกสารประกอบ และคู่มือการติดตั้งผ่านพื้นที่เก็บข้อมูลโครงการ แพลตฟอร์มต้องการบัญชี AWS โดเมนที่กำหนดค่าใน Route53 และความคุ้นเคยพื้นฐานกับบริการ AWS ไม่จำเป็นต้องมีความเชี่ยวชาญ DevOps เฉพาะทาง การติดตั้งอัตโนมัติจัดการการกำหนดค่าเครือข่าย ความปลอดภัย การติดตามสถานะ และการสำรองข้อมูล

เมื่อการพัฒนาที่ได้รับความช่วยเหลือจาก AI กลายเป็นพื้นฐานสำหรับทีมซอฟต์แวร์ที่มีการแข่งขัน ต้นทุนและความซับซ้อนยังคงเป็นอุปสรรคสำคัญสำหรับองค์กรขนาดเล็ก แพลตฟอร์ม AWS Code-Server Multi-Developer แสดงให้เห็นว่าโครงสร้างพื้นฐานและความสามารถ AI ระดับองค์กรสามารถเข้าถึงได้สำหรับทีมทุกขนาด โดยการคิดใหม่เกี่ยวกับรูปแบบสภาพแวดล้อมการพัฒนาบนคลาวด์รอบๆ การใช้ทรัพยากรร่วมกันอย่างมีประสิทธิภาพและบริการ AI แบบจัดการ แพลตฟอร์มส่งมอบเครื่องมือระดับมืออาชีพในราคาที่เหมาะกับสตาร์ทอัพ

## เกี่ยวกับแพลตฟอร์มนี้

แพลตฟอร์ม AWS Code-Server Multi-Developer เป็นโซลูชัน Infrastructure-as-Code แบบเปิดที่สร้างด้วย AWS CDK Python ผสมผสาน code-server (VS Code บนเบราว์เซอร์), AWS Bedrock (Claude AI แบบจัดการ) และบริการโครงสร้างพื้นฐาน AWS เพื่อส่งมอบสภาพแวดล้อมการพัฒนาบนคลาวด์ที่คุ้มค่า แพลตฟอร์มออกแบบสำหรับทีมพัฒนา 2-8 คนที่ต้องการสภาพแวดล้อมการพัฒนาบนคลาวด์พร้อมการช่วยเหลือ AI แบบบูรณาการโดยไม่ต้องรับมือกับความซับซ้อนและต้นทุนของโซลูชัน VDI แบบดั้งเดิม

## ข้อมูลจำเพาะทางเทคนิค

- **โครงสร้างพื้นฐาน**: AWS CDK Python 2.x ติดตั้งได้ในทุกภูมิภาค AWS
- **การประมวลผล**: เซิร์ฟเวอร์ EC2 เครื่องเดียว (ค่าเริ่มต้น t3.2xlarge) พร้อมคอนเทนเนอร์ Docker แยกกัน 8 ตัว
- **พื้นที่จัดเก็บข้อมูล**: ไดรฟ์ EBS 500GB พร้อมการสำรองข้อมูลอัตโนมัติรายวัน (เก็บไว้ 30 วัน)
- **เครือข่าย**: VPC, Application Load Balancer พร้อม SSL/TLS, Route53 DNS
- **การผสานรวม AI**: AWS Bedrock พร้อม Claude 3 Haiku, Sonnet, 3.5 Sonnet และ Sonnet 4.5
- **ความปลอดภัย**: การยืนยันตัวตนตาม IAM role, security groups, secrets แบบเข้ารหัส
- **การติดตามสถานะ**: CloudWatch Logs, Metrics, Alarms และการติดตามการใช้งาน
- **เวลาติดตั้ง**: ประมาณ 30 นาทีจากเริ่มต้นจนถึงสภาพแวดล้อมพร้อมใช้งาน

## โครงสร้างต้นทุน

**โครงสร้างพื้นฐาน (คงที่)**: ~420-450 เหรียญสหรัฐต่อเดือนสำหรับนักพัฒนา 8 คนในภูมิภาคกรุงเทพฯ

- EC2 t3.2xlarge: 273 เหรียญสหรัฐต่อเดือน
- พื้นที่จัดเก็บข้อมูล EBS 500GB: 48 เหรียญสหรัฐต่อเดือน
- Application Load Balancer: 27 เหรียญสหรัฐต่อเดือน
- CloudWatch, สำรองข้อมูล, เครือข่าย: ~70 เหรียญสหรัฐต่อเดือน

**AWS Bedrock (ผันแปร)**: ตามการใช้งาน AI จริง

- Claude 3 Sonnet: 3 เหรียญสหรัฐอินพุต / 15 เหรียญสหรัฐเอาต์พุต ต่อ 1 ล้านโทเค็น
- Claude 3 Haiku: 0.25 เหรียญสหรัฐอินพุต / 1.25 เหรียญสหรัฐเอาต์พุต ต่อ 1 ล้านโทเค็น

**ตัวอย่างต้นทุนรวม**:

- การใช้งาน AI เบา (1 ล้านโทเค็น/คน/เดือน): ~570 เหรียญสหรัฐต่อเดือนรวม (~71 เหรียญสหรัฐ/คน)
- การใช้งาน AI ปานกลาง (10 ล้านโทเค็น/คน/เดือน): ~1,860 เหรียญสหรัฐต่อเดือนรวม (~232 เหรียญสหรัฐ/คน)
- การใช้งาน AI หนัก (50 ล้านโทเค็น/คน/เดือน): ~7,620 เหรียญสหรัฐต่อเดือนรวม (~952 เหรียญสหรัฐ/คน)

**การประหยัดเมื่อเทียบกับโซลูชันแบบดั้งเดิม**: ลดต้นทุนโครงสร้างพื้นฐาน 40-60% เมื่อเทียบกับเดสก์ท็อปคลาวด์แยกกันบวกกับการสมัครใช้งาน AI แยกกัน

## ความพร้อมใช้งาน

แพลตฟอร์ม AWS Code-Server Multi-Developer พร้อมใช้งานแล้วสำหรับการติดตั้งในภูมิภาค AWS ที่รองรับ VPC และ EC2 การผสานรวม AWS Bedrock ต้องการการกำหนดค่าข้ามภูมิภาคสำหรับทีมในภูมิภาคที่ไม่มี Bedrock รวมเอกสารประกอบฉบับสมบูรณ์ คู่มือการติดตั้ง และซอร์สโค้ดไว้ด้วย

## เริ่มต้นใช้งาน

1. โคลนพื้นที่เก็บข้อมูลและเข้าไปยังไดเรกทอรี `cdk/`
2. แก้ไข `config/prod.py` ด้วยโดเมน, Route53 zone ID และ IP ผู้ดูแลระบบของคุณ
3. รัน `bash scripts/deploy.sh` หรือ `cdk deploy --all`
4. เปิดใช้งานการเข้าถึงโมเดล Claude ใน AWS Bedrock Console (ภูมิภาคสิงคโปร์)
5. SSH ไปยัง EC2 ดึงรหัสผ่านจาก Secrets Manager เริ่มคอนเทนเนอร์ Docker
6. เข้าถึงสภาพแวดล้อมที่ dev1-dev8.yourdomain.com

คำแนะนำโดยละเอียดมีอยู่ใน `cdk/QUICKSTART.md` (คู่มือ 5 นาที) และ `cdk/README.md` (เอกสารประกอบครบถ้วน)
